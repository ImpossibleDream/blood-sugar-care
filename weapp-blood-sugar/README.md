## 上传微信开发者工具步骤

### 第一步：获取项目

1. 打开 **微信开发者工具**
2. 点击 **项目 → 导入项目**
3. 选择目录：`weapp-blood-sugar/`
4. 填写 **AppID**（可使用测试号）
5. 点击 **导入**

### 第二步：配置云开发环境

1. 在微信开发者工具顶部工具栏，点击 **云开发** 图标
2. 开通云开发（如未开通）
3. 创建云环境，记下 **环境ID**（如 `blood-sugar-xxx`）
4. 在开发者工具中，右键 `cloudfunctions/login` → **上传并部署：云端安装依赖**

### 第三步：修改配置

1. 打开 `app.js`，修改 `wx.cloud.init` 中的 `env` 为你的云环境ID：

```javascript
wx.cloud.init({
  env: '你的环境ID',  // ← 替换这里
  traceUser: true
})
```

2. 打开 `project.config.json`，将 `appid` 替换为你的小程序 AppID：

```json
"appid": "wx你的AppID"  // ← 替换这里
```

### 第四步：创建云数据库集合

在云开发控制台的 **数据库** 中，创建以下集合：

| 集合名 | 说明 | 权限 |
|--------|------|------|
| `blood_sugar_records` | 血糖记录 | 所有用户可读，仅创建者可写 |
| `blood_sugar_injections` | 注射记录 | 所有用户可读，仅创建者可写 |
| `blood_sugar_users` | 用户登录日志 | 仅创建者可读写 |

### 第五步：运行

1. 在微信开发者工具中，点击 **编译**
2. 如果不需要云开发，应用会自动降级为**本地缓存模式**

## 云开发环境配置教程

### 首次配置（详细步骤）

1. **开通云开发**
   - 微信开发者工具 → 顶部工具栏 → 「云开发」按钮
   - 点击「开通」，选择环境名称（如 `blood-sugar-prod`）
   - 等待环境创建完成

2. **部署云函数**
   - 展开 `cloudfunctions/` 目录
   - 右键 `login` → 「上传并部署：云端安装依赖」
   - 等待上传完成，状态变为「已部署」

3. **创建数据库集合**
   - 云开发控制台 → 数据库 → 「+ 创建集合」
   - 分别创建三个集合（集合名见上表）
   - 每个集合点击「权限设置」→ 选择「自定义安全规则」
   - 推荐基础规则（根据实际需求调整）：

```json
// blood_sugar_records 安全规则
{
  "read": true,
  "write": "doc._openid == auth.openid"
}
```

### 演示模式（无需云开发）

如果没有配置云环境，小程序会自动使用**本地缓存模式**：
- 数据存储在 `wx.setStorageSync`
- 预加载演示数据（与 HTML 文件一致）
- 所有 CRUD 操作在本地完成
- 图表数据自动从本地缓存计算

## 功能说明

### 登录双模式

| 模式 | 方式 | 说明 |
|------|------|------|
| 微信授权 | `getUserInfo` + 云函数 | 获取微信昵称/头像 + OpenID |
| 手机验证码 | 手机号 + 验证码 | 带倒计时 UI，支持模拟验证 |

### 数据看板

| 图表 | 类型 | 说明 |
|------|------|------|
| 月度血糖均值趋势 | 折线图 | 7个时段按月均值变化 |
| 双轨预警对比 | 柱状图 | 个人阈值 vs 医学标准达标率 |
| 各时段预警占比 | 环形图 | 偏高记录按时段分布 |
| 注射剂量 vs 血糖 | 组合图 | 柱状(剂量) + 折线(血糖) |
| 各时段血糖分布 | 箱线图 | 分布范围、中位数、分位数 |
| 极值记录 | 散点图 | Top5极值标注 |

### 数据操作

- **新增**: 选择日期/时段/数值 → 添加
- **编辑**: 点击表格中「编辑」按钮 → 修改 → 保存
- **删除**: 点击「删除」→ 确认后删除
- **筛选**: 按日期范围、时段过滤
- **导出**: CSV 格式导出（UTF-8 BOM）
- **报告**: 按日期范围生成文本报告
